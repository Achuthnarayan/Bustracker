# 🔧 Quick Fix for 404 Errors

## The Problem

You're seeing 404 errors for `favicon.ico` and `index` files on Vercel.

## The Solution (2 minutes)

### Step 1: Update Your Code

```bash
cd Bustracker

# Add the updated files
git add .
git commit -m "Fix 404 errors - update vercel.json"
git push
```

Vercel will automatically redeploy in 2-3 minutes.

### Step 2: Wait for Redeploy

Watch your Vercel dashboard - it should show "Building..." then "Ready"

### Step 3: Test

Visit: `https://your-app.vercel.app`

You should now see the login page!

## What Was Fixed

I updated `vercel.json` to properly route requests:
- `/` → `/tracking/index.html`
- `/api/*` → API functions
- Everything else → `/tracking/*`


### Option 2: Redeploy via Dashboard

1. Go to Vercel dashboard
2. Click your project
3. Click "Deployments"
4. Click "..." on latest deployment
5. Click "Redeploy"

## Common Causes

### Cause 1: Files Not Committed

**Check:**
```bash
git ls-files tracking/
```

Should show all your HTML, CSS, JS files.

**Fix:**
```bash
git add tracking/
git commit -m "Add tracking files"
git push
```

### Cause 2: Wrong Root Directory

**Check:** Vercel Settings → General → Root Directory

**Should be:** `./` or blank

**Fix:** Update in Vercel dashboard, then redeploy

### Cause 3: Build Failed

**Check:** Vercel dashboard → Deployments → Click latest → View logs

**Look for:** Red error messages

**Common errors:**
- "Cannot find module" → Run `npm install` in root
- "ENOENT" → File missing
